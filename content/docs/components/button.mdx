# Button & ButtonLink

TsxForge provides an enhanced `<Button />` and `<ButtonLink />` that wrap the underlying standard components with additional utility props for typical application needs.

## Button

The `Button` component builds upon standard button styling with built-in loading states and content slots, making it easier to build complex buttons without additional wrapper elements.

### Usage

### Custom Props

| Prop | Type | Default | Description |
| ---- | ---- | ------- | ----------- |
| `isLoading` | `boolean` | `false` | When true, renders a spinner in place of `startContent` and disables the button. |
| `startContent` | `React.ReactNode` | `undefined` | Content to display before the children (typically an icon). |
| `endContent` | `React.ReactNode` | `undefined` | Content to display after the children (typically an icon). |
| `classNames` | `{ startContent?: string, endContent?: string }` | `undefined` | Custom classes for the specific content slots. |
| `asChild` | `boolean` | `false` | Whether the button should render as its custom child component (using Radix `Slot`). |

## ButtonLink

The `ButtonLink` component combines the `Button` styling options with TanStack Router's `Link` functionality.

### Usage

```tsx
import { ButtonLink } from "@/components/tsxforge/button-link"

export default function Example() {
  return (
    <ButtonLink to="/profile" params={{ id: "123" }} variant="outline">
      Go to Profile
    </ButtonLink>
  )
}
```

### Custom Props

`ButtonLink` inherits all the routing capabilities from TanStack Router, plus the styling and capabilities of the `Button`.

| Prop | Type | Default | Description |
| ---- | ---- | ------- | ----------- |
| Standard TanStack `Link` | - | - | e.g. `to`, `params`, `search`, etc. |
| `isLoading` | `boolean` | `false` | Displays a loading spinner and prevents navigation clicks when true. |
| `startContent` | `React.ReactNode` | `undefined` | Content to display before the label. |
| `endContent` | `React.ReactNode` | `undefined` | Content to display after the label. |
