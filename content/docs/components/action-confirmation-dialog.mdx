# Action Confirmation Dialog

The `ActionConfirmationDialog` simplifies the implementation of critical actions where user certainty must be established via a popup overlay. Instead of manually coordinating `AlertDialog` triggers, contents, descriptions, cancels, and actions respectively, it wraps the syntax entirely taking in semantic `title`, `description`, and behavior strings.

## Features

- Employs ShadCN UI's `AlertDialog` primitives for full accessible standards including `focus-trapping` and correct `aria-describedby`/`aria-labelledby` semantics.
- Visual icon placements inside the header geometry representing danger/confirmation schemas.
- Wraps triggering children using standard `asChild` behaviors reducing extraneous DIV injections.

## Custom Props

| Prop | Type | Default | Description |
| ---- | ---- | ------- | ----------- |
| `title` | `string` | `"Are You sure"` | Broad header title. |
| `description` | `string` | `"This action can not be undone"` | Informational warning text. |
| `action` | `() => void` | **Required** | Executable callback triggered sequentially following "Confirm" click. |
| `confirmText` | `string` | `"Confirm"` | Verbiage for acceptance button label. |
| `cancelText` | `string` | `"Cancel"` | Verbiage for returning cancel button label. |
| `icon` | `IconSvgElement` | `Alert01Icon` | Header SVG descriptor. |
| `asChild` | `boolean` | `false` | Whether to behave as a semantic Radix trigger pushing properties to the `children` boundary. |
