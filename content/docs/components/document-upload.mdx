# DocumentFileUpload

The `DocumentFileUpload` component provides a robust interface for uploading documents. It features drag-and-drop support, file size validation, and a clear preview of the selected file.

## Features

- **Drag & Drop**: Intuitive file uploading with visual dragging states.
- **Validation**: Built-in checks for file size and accepted formats.
- **File Preview**: Displays the name and size of the selected file with an option to remove it.
- **Ref API**: Programmatic control to clear or retrieve the current file.

## Custom Props

| Prop | Type | Default | Description |
| ---- | ---- | ------- | ----------- |
| `onFileChange` | `(file: File \| null) => void` | `undefined` | Callback triggered when a file is selected or removed. |
| `maxSize` | `number` | `5242880` (5MB) | Maximum allowed file size in bytes. |
| `accept` | `string` | `".pdf,.doc,.docx,.jpg,.jpeg,.png,.webp"` | Allowed file extensions or MIME types. |
| `disabled` | `boolean` | `false` | Whether the uploader is interactive. |
| `initialFile` | `File \| null` | `null` | Pre-populate the uploader with a `File` object. |
| `initialUrl` | `string \| null` | `null` | Display an existing file via its URL. |

## Ref API

The component exposes the following methods via `useImperativeHandle`:

```tsx
const ref = useRef<DocumentFileUploadRef>(null);

// Clear the currently selected file
ref.current?.clearFile();

// Retrieve the currently selected File object
const file = ref.current?.getFile();
```
